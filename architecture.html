Telegram Commands

/update

/portfolio
포트폴리오 리포트 조회. 기준가, 수익률(1D~YTD), 기여도 Top5. NAV + 수익률을 먼저 계산한 뒤 리포트 생성.
calculate_wrap_nav.py → calculate_returns.py → daily_portfolio_report.py → 텔레그램 전송

Data Files

Wrap_NAV.xlsx
마스터 데이터. 포트폴리오별 종목/비중/기준가 저장.
시트: NEW(날짜·증권사·상품명·업종·코드·종목·비중), 기준가(일별NAV), 수익률(기간별), Code(섹터매핑)
로컬 수정 → 저장 시 watch_wrap_nav.py가 자동 git push → VM이 16:00에 pull하여 계산에 반영
portfolio_data.json
실시간 종목별 오늘수익률, 기여도, 누적수익률.
/update 실행 시 갱신. Dashboard 테이블 데이터 소스.
index.html
GitHub Pages 메인 대시보드. 시장 차트(지수/환율/금리/암호화폐/메모리/원자재) + 상단 네비게이션(WRAP / 투자유의종목 / Architecture).
git push 시 자동 배포.
wrap.html
WRAP 전용 페이지. 비밀번호 보호 (SHA-256 클라이언트 검증, sessionStorage 세션 유지).
WRAP 기준가 차트 + 수익률 비교 테이블(RETURN, 1D~YTD) + Portfolio 테이블 + Sector Weight 비교.
create_dashboard.py에서 생성. GHA file_pattern + weather_bot.py git add에 포함.
market_alert.html
투자유의종목 현황 페이지. 투자위험 → 투자경고 → 투자주의 순서.
투자주의: 지정유형·경고전환가(min(5일전×1.6, 15일전×2.0)) 표시, 투자경고 지정예고 행 강조(#fef2f2).
투자경고/위험: 판단일·현재가·15일최고가·해제가능주가, 2단계 배경색(판단일 도달/#fca5a5, 5영업일 이내/#fee2e2).
KIND POST API 스크래핑. FDR KRX-DESC fallback + 네이버 시가총액 보충. VM 15:40 / 23:00 자동 재생성 (timeout 180초).

Troubleshooting

증상 원인 확인 방법
/update 보냈는데 수익률이 안 바뀜 create_portfolio_tables.py 에러로 실패 sudo journalctl -u weather_bot -n 50
텔레그램 봇이 응답 없음 봇 서비스 중단됨 sudo systemctl status weather_bot
sudo systemctl restart weather_bot
Dashboard가 업데이트 안 됨 git push 실패 (인증 만료 등) cd ~/Antigravity_Market_Dashboard && git push
날짜 타입 에러 (str vs datetime) Wrap_NAV.xlsx 날짜 컬럼 타입 불일치 pd.to_datetime() 적용 확인
GitHub Actions 실패 크롤링 대상 사이트 변경 등 GitHub → Actions 탭 → 실패 로그 확인
VM SSH 접속 안 됨 키 파일 권한 또는 보안 그룹. OCI는 cloud-init이 부팅 시 authorized_keys를 메타데이터 키로 덮어씀 → Boot volume repair로 수동 수정해도 재부팅하면 원래대로 돌아옴 ssh -i "C:\Users\user\Antigravity_Market_Dashboard\ssh-key-2026-02-20.key" ubuntu@144.24.70.224
