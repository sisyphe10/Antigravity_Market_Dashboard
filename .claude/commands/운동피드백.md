# 선유듀오 운동봇 피드백 팁 추가

선유듀오 운동봇(@SeonyuDuo_bot)이 운동 리마인드(캘린더 1시간 전 / `/remind`)에 유형별로 **랜덤 3개씩** 붙여주는 "📝 피드백" 팁을 추가하는 커맨드.

- 데이터: `seonyuduo_feedback_tips.json` (레포 루트). 키 = 운동 유형(`테니스`/`러닝`/`워크아웃`), 값 = 팁 문자열 배열.
- 봇 코드: `execution/seonyuduo_exercise_bot.py`
  - `_load_feedback_tips()` (~L944) — 매 호출마다 디스크에서 새로 읽음(코드 수정 없이 JSON만 편집하면 반영).
  - `_format_feedback_tips(유형, n=FEEDBACK_TIPS_N)` (~L956) — `random.sample`로 3개 추출 → `• {팁}` 렌더.
  - 리마인드 메시지에 연결 (~L1367).
- **순서는 무관**(랜덤 추출) → 새 팁은 해당 유형 배열 끝에 append하면 충분.

## 사용법
- 한 개: `/운동피드백 포핸드 백핸드 / 타점을 훨씬 앞에 두고 힘으로 치지 말고 플랫하게`
- 여러 개(줄바꿈으로 한 번에):
  ```
  /운동피드백
  백발리 / 왼손 테이크백 몸 앞에 두기
  포발리 / 검지로 라켓 엎지 말고 슬라이스성으로 버티기
  ```
- 유형 지정: 기본 `테니스`. 러닝/워크아웃이면 팁 앞에 `[러닝]` / `[워크아웃]` 태그(예: `[러닝] 케이던스 180까지`).

## 팁 형식 규칙 (`$ARGUMENTS` 정규화)
각 팁은 `"{카테고리} / {내용}"` 문자열. 테니스 카테고리는 봇의 `TENNIS_CATEGORIES` 7개로 표준화:
`포핸드`, `백핸드`, `슬라이스`, `포핸드 발리`, `백핸드 발리`, `서브`, `공통`

별칭 정규화:
- `포발리` → `포핸드 발리`, `백발리` → `백핸드 발리`
- `발리`(단독) → 그대로 `발리` (기존 데이터에 다수 존재 → 유지)
- `포핸드 백핸드` / `포백`(둘 다 해당) → `포핸드 / 백핸드` (기존 `포핸드 발리 / 백핸드 발리 / …` 선례와 동일하게 카테고리를 ` / `로 나열)
- **내용 문구는 사용자 원문 그대로** 보존. 카테고리 라벨만 표준화한다.
- 카테고리가 불명확하면 추측하지 말고 사용자에게 확인(또는 `공통` 제안).

## 워크플로
0. **현재 파일 Read** — `seonyuduo_feedback_tips.json` 전체를 읽어 기존 항목/형식 확인. **중복(동일 문장)이면 추가 보류**하고 사용자에게 알림.
1. **정규화** — 위 규칙으로 각 팁을 `"{카테고리} / {내용}"`로 변환.
2. **추가** — 해당 유형 배열 끝에 Edit로 항목 추가. 기존 항목 보존, JSON 형식 유지(직전 마지막 항목 뒤 콤마 주의). 대상 유형 풀이 비어 있으면(`워크아웃: []`) 첫 항목 형식 맞춰 삽입.
3. **검증** — JSON parse + 유형별 항목 수 확인:
   ```bash
   PY=/c/Users/user/anaconda3/envs/myenv/python.exe
   PYTHONIOENCODING=utf-8 "$PY" -c "import json;d=json.load(open('seonyuduo_feedback_tips.json',encoding='utf-8'));print({k:len(v) for k,v in d.items()})"
   ```
   봇 syntax 1회(변경 없어도 안전):
   ```bash
   PYTHONIOENCODING=utf-8 "$PY" -c "compile(open('execution/seonyuduo_exercise_bot.py',encoding='utf-8').read(),'b','exec');print('OK')"
   ```
4. **배포** (★사용자 승인 후) — 먼저 VM 실제 KST 확인, **16:00~17:00 KST 배포 금지**(sisyphe-bot cron race, deploy.sh가 전 봇 restart):
