# Antigravity Market Dashboard - Claude 작업 규칙

## 답변 규칙
- 모든 답변은 한국어로 작성 (일본어, 불필요한 영어 혼용 금지)
- 완료 후 불필요한 요약 반복하지 않기
- 에러 발생 시 원인 + 재발 방지까지 설명

## 작업 프로세스 (필수)
모든 작업은 반드시 아래 순서를 따른다:
1. **계획** — 무엇을 왜 어떻게 할지 먼저 설명하고 승인받기
2. **실행** — 코드 작성/수정
3. **검증** — 실행 결과 확인, 사이드이펙트 점검, 데이터 정합성 체크
4. **보고** — 검증 결과를 포함하여 완료 보고

검증 없이 "완료했습니다"라고 보고하지 않는다.

## 자율 조사/해결 원칙
- 사용자의 권한이 필요 없고 직접 확인/해결할 수 있는 문제는 사용자에게 시키지 말고 스스로 조사하고 해결한다
- 예: API 응답 확인, 로그 분석, 코드 디버깅, 인덱스 매핑 검증 등은 WebFetch/Grep/Read 등으로 직접 수행
- 브라우저 콘솔 확인, 요소 검사 등 사용자만 할 수 있는 작업만 요청한다

## 검증 체크리스트 (보고 전 필수)
- 코드 변경 → 실행해서 에러 없는지
- 데이터 변경 → 값이 기대한 대로인지 (직접 출력해서 확인)
- git 작업 → `git status`로 상태 확인, 파일 소실 없는지
- push → 성공 여부 확인
- 사용자가 "안 보인다/안 된다"고 하면 → 캐시/새로고침 탓하지 말고 **코드 버그부터 확인**

## 변경 시 사이드이펙트 점검
- dataset.csv에 새 데이터 타입 추가 시: draw_charts.py, create_dashboard.py 등 dataset.csv를 읽는 모든 파일에서 필터링 확인
- UI 요소 변경 시: 모든 페이지에서 일관성 확인

## Git 규칙
- `git pull --rebase` 절대 사용 금지
- `git reset --hard` 로컬에서 사용 금지
- Wrap_NAV.xlsx 충돌 시: merge에서 `--ours` = 로컬
- GHA 자동커밋 때문에 push 실패는 일상적. push → pull → 충돌 해결 → 재생성 → push가 기본 패턴

## Windows 환경
- Python 실행 시 항상 `PYTHONIOENCODING=utf-8` 설정
- 항상 절대경로 사용하거나, 실행 전 프로젝트 디렉토리로 이동
- matplotlib 차트에서 유니코드 특수문자 사용 금지, 영문/한글/기본 기호만

## 코드 작성 규칙
- Python: PEP8, UTF-8
- 비밀 정보(API 키 등)는 .env 파일로 분리
