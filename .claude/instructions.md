# 개발 환경 및 답변 규칙

## 기본 원칙
- 모든 답변과 주석은 한글로 작성
- 초보자도 이해할 수 있게 자세히 설명
- 코드 실행 전 반드시 무엇을 하는지 설명
- 에러 발생 시 해결 방법까지 함께 제시

## 코드 작성 규칙
- Python 코드는 PEP8 스타일 가이드 준수
- 모든 함수와 클래스에 한글 주석 추가
- 변수명은 영어로 하되, 주석으로 설명 추가
- 파일은 항상 UTF-8 인코딩으로 저장

## 프로젝트 특화 사항
- 웹 크롤링: requests, BeautifulSoup4, Selenium 사용
- 웹페이지: 간단한 HTML/CSS/JavaScript 또는 Flask 사용
- 텔레그램 봇: python-telegram-bot 라이브러리 사용
- 모든 비밀 정보(API 키 등)는 .env 파일로 분리

## 파일 구조
- 프로젝트마다 requirements.txt 생성
- README.md에 실행 방법 상세히 작성
- .gitignore 자동 생성 (API 키, 환경 변수 보호)

## 에러 처리
- try-except 구문 적극 활용
- 로그 메시지는 한글로 작성
- 에러 발생 시 사용자 친화적 메시지 출력
